Unit Testing the app

User Acceptance test

Tester1: Relative of a team member

Objective/Testing purpose: Testing a treasure hunt sample game and assessing general design and UI

Results: User commented on the design, mentioning that it is appealing and decent.
However, the user also mentioned that the app is buggy, and the responsiveness can be laggy and frustrating.

Overall: User had a decent experience.




Tester2: UCLan Cyprus Student

Objective/Testing purpose: Evaluating design and functionality

Results: According to the user, the design is good and the color palette/choice was a nice touch.
User also commented on the leaderboard, saying that it's broken and suggested we remove it.
User tested functionality, did not mention anything except the QR camera, which opens the front cam instead of the back.

Overall: User had a good experience.




Tester3: Friend of a team member

Objective/Testing purpose: Testing a treasure hunt sample game and assessing general design and UI

Results: User described the app as "basic" and "functional", and describes the experience as "acceptable".

Overall: User had a decent experience.
